# 【每日大模型行业情报简报】

**日期：2026年3月7日**
**编辑：AI 行业投资顾问**

---

### 01. 核心摘要

今日数据显示，大模型行业呈现明显的“量价背离”与“代际更替”特征。**Minimax** 凭借 `minimax-m2.5` 在Token消耗量上断层领先，显示出极强的C端或应用层渗透率；**Anthropic** 在LMArena跑分榜单上重夺王座，`Claude Opus 4.6` 性能登顶；**Google** 新一代 `Gemini 3` 系列表现强势，在消耗量与跑分两端均对 OpenAI 构成挤压态势。OpenAI 在消耗榜单前五缺席，仅靠 `gpt-oss-120b` 与 `gpt-5-nano` 维持份额，面临较大的市场竞争压力。

---

### 02. 核心厂商动态与用量分析

#### 2.1 Minimax：用量断层第一，应用生态爆发
*   **数据表现**：`minimax/minimax-m2.5` 近7天 Token 消耗高达 **1981.51 Billion**，远超第二名 Google Gemini 3 Flash 约 60%。
*   **分析师点评**：Minimax 已成为当前流量最大的模型厂商之一。如此巨大的消耗量通常意味着其背后有极高 DAU 的 C 端应用（如海螺AI等）或大规模 B 端集成。`m2.5` 模型显然在处理高并发、低成本请求上具有极大优势，市场占有率目前处于统治地位。

#### 2.2 Google：Gemini 3 代际跃升，攻势凌厉
*   **数据表现**：`google/gemini-3-flash-preview` 以 **1239.22 Billion** 的消耗量位居第二。同时，老一代 `gemini-2.5-flash` 仍有 593.33 Billion 的存量。
*   **分析师点评**：Google 的模型迭代速度极快，用户从 2.5 向 3.0 迁移的迹象明显。Gemini 3 Flash 的高消耗量验证了其在多模态处理与长文本场景下的竞争力，正在快速蚕食原本属于 GPT 系列的中端市场份额。

#### 2.3 中国厂商群像：Kimi、DeepSeek 稳居第一梯队
*   **Moonshot (Kimi)**：`kimi-k2.5` 消耗量 **941.40 Billion**，排名第三。Kimi 在长文本与 Agent 场景的深耕使其保持了极高的用户粘性。
*   **DeepSeek**：`deepseek-v3.2` 消耗量 **892.02 Billion**，排名第四。DeepSeek 持续以高性价比策略维持开发者生态的高活跃度。
*   **Stepfun (阶跃)**：`step-3.5-flash:free` 消耗量 **675.52 Billion**。作为榜单中少见的“免费/Free”标签模型，阶跃星辰通过免费策略成功换取了巨大的流量份额，排名第七。

#### 2.4 Anthropic：高端模型统治力依旧
*   **数据表现**：`claude-opus-4.6` (872.90B) 与 `claude-sonnet-4.6` (800.68B) 分别位列第五、六名。
*   **分析师点评**：尽管价格通常高于 Flash 类模型，但 Anthropic 的 Opus 和 Sonnet 系列依然保持了极高的消耗量，说明其在复杂任务、代码生成等高价值场景中拥有不可替代的地位。

#### 2.5 OpenAI：份额承压，策略调整
*   **数据表现**：OpenAI 未进入消耗榜前五。`gpt-oss-120b` (444.06B) 和 `gpt-5-nano` (287.83B) 分别位列第 13、15 位。
*   **分析师点评**：OpenAI 目前的流量主力似乎已从通用大模型转向特定细分模型（如 nano 系列），但在总量上已被 Minimax、Google、Moonshot 等反超。这反映出 OpenAI 在 API 市场面临严峻的“左右互搏”与外部竞争。

### 03. 跑分榜单与技术前沿

根据 LMArena 最新排名，行业格局发生重大变动：

*   **Anthropic 重夺第一**：`Claude Opus 4.6` 以 **1504 分** 登顶，其 Thinking 版本紧随其后（1502分）。这标志着 Claude 在推理能力上再次超越对手。
*   **Google Gemini 3 紧随其后**：`Gemini 3.1 Pro Preview` 以 **1500 分** 位列第三，`Gemini 3 Pro` (1485分) 和 `Gemini 3 Flash` (1473分) 均表现优异。Google 的新一代模型在性能上已完全对标甚至超越 GPT 系列。
*   **OpenAI 落后半个身位**：`GPT-5.4-high` 得分 **1479 分**，排名低于 Claude Opus 4.6 和 Gemini 3.1 Pro。这是近年来 OpenAI 在跑分榜单上罕见的落后局面，显示其技术护城河正在变窄。
*   **xAI 崛起**：`Grok-4.20-beta1` (1491分) 进入前五，显示出 xAI 在模型训练上的巨大进步。

---

### 04. 多模态专区

*   **Vision 模型动态**：
    虽然本次数据库采样中未单独列出 Vision 跑分子榜，但从 Token 消耗来看，`Gemini 3 Flash` 与 `Minimax M2.5` 的高消耗量极大概率包含大量多模态（图文/视频）处理请求。
*   **开源多模态价格下探**：
    定价数据显示，`allenai/molmo-2-8b` 的 API 价格已下探至 **Input $0.19 / Output $0.19** (每百万 Token)。Molmo-2 作为知名开源多模态模型，其低廉的推理成本将进一步降低多模态应用的开发门槛。
*   **分析**：随着 Gemini 3 原生多模态能力的普及，以及 Molmo-2 等开源模型的低价竞争，纯文本模型的市场空间将被进一步压缩。建议投资者关注具备“原生多模态”能力且成本控制优秀的厂商。

---

### 05. 价格战与性价比观察

*   **阿里通义**：`tongyi-deepresearch-30b-a3b` 展现了极具攻击性的定价策略，输入价格低至 **$0.088/M**，输出 **$0.449/M**，且拥有高达 **93%** 的缓存命中率。这表明阿里正在通过极致的架构优化（如 A3B 架构）争夺长尾开发者市场。
*   **免费策略**：榜单中 `stepfun/step-3.5-flash:free` 和 `arcee-ai/trinity-large-preview:free` 的高消耗量证明，“免费”依然是获取市场份额的有效手段，尤其是在对延迟不敏感但对成本极度敏感的场景中。

---

### 06. 投资建议

1.  **关注应用层变现**：Minimax 和 Kimi 的高消耗量证明了模型能力向应用层转化的成功，建议重点关注具备“模型+应用”闭环能力的厂商。
2.  **警惕 OpenAI 龙头松动**：OpenAI 在跑分和消耗量上的双重相对下滑是一个重要信号，需密切关注其后续 GPT-5 系列完整版的发布情况。
3.  **多模态红利期**：随着 Gemini 3 和 Molmo-2 的推进，多模态推理成本大幅下降，建议布局基于多模态理解的视频分析、具身智能等下游应用。
