### **一、 核心观点摘要**

今日数据显示，大模型行业竞争格局出现显著分层。**Anthropic** 凭借 Claude Opus 4.6 登顶 LMArena 榜首，在代码与综合能力上全面超越 OpenAI。**Minimax** 异军突起，其 `minimax-m2.5` 模型以惊人的 1855B Token 消耗量霸榜，显示出极强的 C 端或应用层渗透率。**Google** Gemini 3 系列在保持高性能的同时，用量激增，成为仅次于 Minimax 的算力消耗大户。相比之下，OpenAI 在榜单头部略显沉寂，GPT-5.2 跑分排名下滑至第六，且 Token 消耗主力未见传统旗舰模型，显示出其市场策略可能正在向更隐蔽或特定版本转移。

---

### **二、 核心厂商动态与模型分析**

#### **1. Anthropic：重回王座，SOTA 模型实至名归**
*   **跑分表现：** `claude-opus-4-6-thinking` 与 `claude-opus-4-6` 以 **1503 分** 并列 LMArena 综合排名第一，且包揽代码能力第一、二名。这是继 Claude 3.5 Sonnet 之后的又一次全面胜利。
*   **市场动量：** `claude-opus-4.6` 与 `claude-sonnet-4.6` 的 Token 消耗量均进入前五（分别为 716B 和 694B），显示出市场对新一代 Claude 模型的付费意愿极强，高定价策略并未阻碍其企业级渗透。

#### **2. Minimax：流量之王，国产模型出海势头迅猛**
*   **数据异动：** `minimax/minimax-m2.5` 以 **1855.13B** 的 Token 消耗量位居榜首，是第二名 Google Gemini 的 1.6 倍。这一数据极其惊人，暗示 Minimax 可能在特定爆款应用（如社交、陪伴类）或海外 API 市场取得了突破性进展。
*   **投资解读：** 这种量级的消耗通常意味着极高的用户粘性或大规模 B 端集成，需重点关注 Minimax 近期的合作伙伴或 C 端产品动态。

#### **3. Google Gemini：高性能与高并存的“性价比”之选**
*   **模型迭代：** `gemini-3.1-pro-preview` 以 1500 分位列第三，紧追 Claude。`gemini-3-flash-preview` 消耗量高达 1154B，排名第二。
*   **策略分析：** Google 采取了“高性能旗舰 + 高吞吐 Flash”的组合拳。Flash 模型在保持较高跑分（1473分，排名第七）的同时，承载了巨大的流量，说明其在成本敏感型应用中极具竞争力。

#### **4. DeepSeek & Moonshot：国产双雄稳居第一梯队**
*   **DeepSeek：** `deepseek-v3.2` 消耗量 908B，排名第三。在保持开源优势的同时，其 API 调用量持续高位，证明其技术壁垒已转化为实实在在的市场份额。
*   **Moonshot (Kimi)：** `kimi-k2.5` 消耗量 768B，排名第四。`MoonshotAIkimi-k2.5-thinking` 在 LMArena 排名第 19，显示出其在长上下文和推理场景的持续投入正在获得市场回报。

#### **5. OpenAI：旗舰跑分承压，流量去向成谜**
*   **跑分预警：** `gpt-5.2-chat-latest` 仅以 1481 分排名第六，已被 Claude 和 Gemini 的新一代模型甩开身位。这是 OpenAI 在榜单头部罕见的“落后”时刻。
*   **流量结构：** 消耗榜前 15 中，OpenAI 仅有一个 `gpt-oss-120b`（362B，排名第 15）。传统主力模型（如 GPT-4/5 系列标准版）未在消耗榜前列出现，推测可能因企业版私有化部署或定价策略导致公开 API 流量占比下降，亦或是用户正在向竞品迁移。

#### **6. 智谱AI (GLM) & 阿里**
*   **智谱：** `glm-5` 以 1455 分位列第 16，消耗量 616B。作为国产模型代表，其在代码能力（Rank 24）和综合能力上均保持稳健。
*   **阿里：** `qwen3.5-397b-a17b` 跑分 1454（第 17 名）。定价方面，`tongyi-deepresearch-30b-a3b` 输入价格低至 **$0.089/M**，输出 **$0.449/M**，显示出阿里正在通过极致的价格战策略争夺中腰部开发者市场。

### **三、 多模态与垂直能力专区**

*   **Vision/多模态动向：**
    虽然今日数据库中未单独列出 Vision 跑分子榜，但值得注意的是，排名前列的 **Gemini 3 系列** 和 **Claude 4.6 系列** 均为原生理多模态模型。Gemini 3 Flash 的高消耗量（1154B）部分归功于其多模态处理能力的下放，使得其在处理图文混合任务时具有极高的性价比。
*   **代码能力：**
    **Anthropic 再次统治代码领域。** `claude-opus-4-6-thinking` 和 `claude-opus-4-6` 占据代码榜前二。相比之下，GPT-5.2 在代码榜仅列第 6，Gemini 3 Pro 甚至跌出前 10（第 11 名）。这表明对于开发者工具而言，Claude 正在成为首选模型。

---

### **四、 价格趋势与成本分析**

*   **价格战白热化：** 阿里 `tongyi-deepresearch` 和 Amazon `nova-lite-v1` 等模型将输入 Token 价格压低至 **$0.06 - $0.09 / Million** 的区间。
*   **对比分析：** 头部闭源模型（如 Claude Opus 4.6）虽然价格高昂（推测定价维持高位），但消耗量依然巨大，说明**“性能优先”**仍是高价值客户的核心诉求；而中长尾模型则完全陷入**“价格敏感”**竞争，输入价格普遍低于 $0.2/M。

### **五、 投资建议与风险提示**

*   **投资建议：**
    1.  **关注 Anthropic 生态：** Claude 在代码和综合推理能力的反超，建议重点关注基于 Claude API 的开发者工具链企业。
    2.  **警惕 OpenAI 增长放缓：** GPT-5.2 跑分落后及公开 API 流量占比下降，提示需重新评估 OpenAI 的技术护城河深度。
    3.  **挖掘 Minimax 产业链：** Minimax 惊人的流量数据背后可能隐藏着下一个“超级应用”，建议追踪其下游主要调用方。

*   **风险提示：**
    *   今日外部搜索资讯（域名论坛等）与 AI 行业关联度极低，数据归因主要依赖内部数据库，可能存在滞后性。
    *   模型迭代速度极快，Gemini 3.1 与 Claude Opus 4.6 的优势窗口期可能较短。
