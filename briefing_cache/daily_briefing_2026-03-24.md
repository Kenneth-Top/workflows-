# 每日大模型行业情报简报
**日期：2026年03月24日**

---

## 1. 核心观点摘要

今日行业呈现**“端侧模型爆发，推理模型登顶”**的双主线特征。
1.  **流量格局重塑**：小米 `mimo-v2-pro` 以近 2 万亿 Token 的周消耗量异军突起，登顶榜首，标志着端侧/手机厂商大模型生态已形成实质性规模。
2.  **推理模型 SOTA 易主**：Anthropic `Claude Opus 4.6 Thinking` 以 1502 分超越 Google 和 OpenAI 最新模型，重夺 LMArena 榜首，推理赛道竞争白热化。
3.  **国产模型高性价比优势延续**：DeepSeek、MiniMax、StepFun 等厂商在消耗量榜单前五中占据三席，验证了国产模型在 API 市场的强劲渗透力。

## 2. 核心厂商动态与流量分析

### 2.1 流量激增与市场格局
根据近 7 天 Token 消耗数据，市场格局出现显著变化，国产模型与端侧模型占据主导地位。

*   **黑马领跑：小米**
    *   **数据表现**：`xiaomi/mimo-v2-pro` 以 **1978.47B** (近 2 万亿) Token 的消耗量位居第一，远超第二名。
    *   **分析师点评**：这一数据极其惊人，推测与小米近期手机端 AI 功能（如智能助手、系统级整合）的大规模放量有关。端侧模型反超云端 API 模型，标志着 AI 落地场景正在从纯 Web/API 向硬件生态转移。

*   **国产第一梯队：StepFun、MiniMax、DeepSeek**
    *   **StepFun (阶跃星辰)**：`step-3.5-flash:free` 消耗量达 **1698.57B**，排名第二。其 "free" 标签表明该流量激增主要由免费策略或大规模推广驱动，用户粘性极高。
    *   **MiniMax**：`minimax-m2.5` 消耗量 **1471.64B**，排名第三。结合榜单末尾的 `minimax-m2.7`，MiniMax 在榜单中占据两席，显示出极强的并发处理能力和市场占有率。
    *   **DeepSeek**：`deepseek-v3.2` 消耗量 **1315.45B**，稳居第四。在缺乏价格数据的背景下，其高流量证明了 DeepSeek 在开发者社区中的刚需地位未变。

*   **海外巨头：Anthropic 与 Google**
    *   **Anthropic**：`claude-sonnet-4.6` (1190.93B) 与 `claude-opus-4.6` (1132.97B) 分列五六名。Opus 作为旗舰模型能有如此高的消耗量，说明其在复杂任务处理上的不可替代性。
    *   **Google**：`gemini-3-flash-preview` (1081.31B) 表现强劲，但旧版 `gemini-2.5-flash` 流量已回落至 663B，显示用户正在快速向 Gemini 3 系列迁移。

### 2.2 跑分榜单：推理之战
LMArena 最新排名显示，具备深度思考 能力的模型已统治榜单。

| 排名 | 模型名称 | 分数 | 关键动态 |
| :--- | :--- | :--- | :--- |
| **1** | **claude-opus-4-6-thinking** | **1502** | **新王登顶**。Anthropic 凭借 Thinking 版本超越 Google，确立推理能力新 SOTA。 |
| **2** | claude-opus-4-6 | 1501 | 基础版本同样强悍，仅落后 Thinking 版本 1 分。 |
| **3** | gemini-3.1-pro-preview | 1493 | Google 最新预览版，紧追其后。 |
| **4** | grok-4.20-beta1 | 1492 | xAI 进步神速，Grok 4.2 系列已进入第一梯队。 |
| **6** | gpt-5.4-high | 1485 | OpenAI 排名略显落后，面临较大竞争压力。 |

**分析师观察**：
*   **Anthropic 完胜**：Claude Opus 4.6 系列包揽前二，且 Thinking 模式成为标配。
*   **Grok 崛起**：xAI 的 `grok-4.20-beta1` 杀入前四，超越了 GPT-5.4-high，显示出马斯克旗下团队在算力与数据上的激进投入。
*   **OpenAI 承压**：GPT-5.4-high 仅列第六，且消耗榜单中未见 OpenAI 模型前列（可能被聚合在 openrouter/hunter-alpha 中，或流量被国产免费模型分流），需警惕其市场份额变化。

## 3. 多模态专区

*   **端侧多模态成为主流**：榜首的 `xiaomi/mimo-v2-pro` 推测为多模态模型，其近 2000B 的消耗量意味着图像理解、OCR 及语音交互已在移动端大规模普及。
*   **Gemini 3 Flash 的多模态渗透**：`gemini-3-flash-preview` 的高消耗量 (1081B) 证明了 Google 在多模态输入（长上下文、视频/图像理解）方面的成本优势正在转化为市场份额。
*   **趋势判断**：纯文本模型的流量增长已放缓，新增流量主要集中在具备 Vision 能力的模型（如 Mimo, Gemini, Claude 系列）。

## 4. API 定价与成本分析

今日采样数据显示，市场呈现两极分化：开源/国产模型极致性价比，闭源/企业级模型高溢价。

*   **性价比之选**：
    *   **Alibaba (通义)**：`tongyi-deepresearch-30b-a3b` 输入价格仅 **$0.089/M**，输出 **$0.449/M**。结合其 40% 的缓存命中率，实际成本极低，适合大规模部署。
    *   **Amazon**：`nova-micro-v1` 输入低至 **$0.034/M**，主打极致低成本场景。

*   **高溢价旗舰**：
    *   **Amazon**：`nova-premier-v1` 输出价格高达 **$12.5/M**，定位高端企业级推理。
    *   **Aion Labs**：`aion-2.0` 输出价格 **$1.59/M**，且缓存命中率高达 92.8%，显示其在特定垂直领域（可能是金融或科研）的高频调用特征。

## 5. 投资建议与风险提示

*   **关注端侧 AI 芯片与生态**：小米 Mimo 模型的爆发验证了“手机+AI”的商业闭环。建议重点关注端侧 NPU 算力升级、内存带宽相关产业链标的。
*   **推理模型竞争加剧**：Anthropic 暂时领先，但 Google 与 xAI 差距极小。对于应用层开发者，建议优先接入 Claude Opus 4.6 Thinking 处理复杂逻辑，接入 Gemini 3 Flash 处理高并发多模态任务以平衡成本。
*   **风险提示**：榜单中大量“免费”模型（如 StepFun）带来的流量激增不可持续，需警惕后续商业化变现不及预期导致的成本收缩风险。
