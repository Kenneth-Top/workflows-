### 🚀 核心厂商动态与用量分析

#### 1. Minimax：流量黑洞，M2.5 霸榜
*   **数据表现**：`minimax/minimax-m2.5` 近 7 天 Token 消耗达 **2172.58 Billion**，位居全榜第一，且大幅领先第二名 Stepfun 约 60%。
*   **投资分析**：Minimax 的高消耗量通常与其在 C 端应用（如海螺 AI）及高并发语音/视频生成场景的调用有关。如此巨大的 Token 量级表明其用户留存与活跃度处于行业巅峰，但也对其推理成本控制提出了极高挑战。

#### 2. Google：Gemini 3 系列全面铺开
*   **数据表现**：`gemini-3-flash-preview` 消耗量达 **1165.33 Billion**，排名第三；同时 `gemini-2.5-flash` 仍保持 629.56 Billion 的高位。
*   **动量解读**：Google 正在快速完成从 2.5 系列向 3.0 系列的流量迁移。Preview 版本即获得千亿级调用，证明其分发渠道（Vertex AI + Google Workspace）极其强势。

#### 3. Anthropic：双旗舰策略稳固
*   **数据表现**：`claude-sonnet-4.6` (963.48B) 与 `claude-opus-4.6` (907.03B) 分列第 5、6 位，两者消耗量旗鼓相当。
*   **投资分析**：Sonnet 与 Opus 的消耗量接近，说明 Anthropic 的用户分层策略成功：重度任务（Coding/写作）用户愿意为 Opus 买单，而日常任务则由 Sonnet 承接，形成了健康的"双引擎"收入结构。

#### 4. DeepSeek 与 Moonshot：国产力量持续输出
*   **DeepSeek**：`deepseek-v3.2` 消耗量 **1128.32 Billion**，稳居第四。在开源与低价策略下，DeepSeek 已成为开发者生态的基础设施。
*   **Moonshot (Kimi)**：`kimi-k2.5` 消耗量 **638.72 Billion**。作为长文本领域的标杆，Kimi 在国内 C 端市场的粘性依然极高。

#### 5. OpenAI：低调的 GPT-5 系列
*   **数据表现**：榜单中未见 GPT-5 主力模型，仅 `gpt-oss-120b` (538.77B) 和 `gpt-5-nano` (370.12B) 入榜。
*   **分析**：OpenAI 可能将主力流量（GPT-5/5.5）通过私有 API 或 ChatGPT 终端直接消化，未完全体现在公开聚合平台的统计中。`nano` 版本的高消耗暗示端侧/轻量级应用正在起量。
