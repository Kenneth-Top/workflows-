**日期：2026年03月01日**

### 📊 核心摘要

今日数据显示，大模型行业竞争格局发生显著位移。**Anthropic** 凭借 Claude Opus 4.6 登顶 LMArena 榜首，在代码与综合能力上超越 GPT 系列与 Gemini。**MiniMax** 异军突起，其 `minimax-m2.5` 模型以近 1.9 万亿 Token 的周消耗量霸榜，显示出极强的 C 端或 B 端渗透率。**Google Gemini 3** 系列在保持高性能的同时，Flash 版本用量激增，验证了其“高性价比”策略的成功。中国厂商方面，**DeepSeek** 与 **Moonshot (Kimi)** 稳居流量第一梯队，而 OpenAI 的 GPT-5.2 在跑分上暂居劣势，且流量未见显性优势，面临严峻挑战。

### 🏢 核心厂商动态与流量分析

#### 1. Anthropic：技术登顶，Opus 4.6 确立新 SOTA
*   **跑分表现**：`claude-opus-4-6-thinking` 与 `claude-opus-4-6` 以 **1503 分** 并列 LMArena 综合排名与代码排名双第一。这标志着 Anthropic 在推理能力上已实现对 OpenAI 的反超。
*   **流量态势**：尽管 Opus 系列定位高端，`claude-opus-4.6` 仍录得 707.9B Token 消耗，排名第六。更值得关注的是 `claude-sonnet-4.6`，以 677.8B 的消耗量紧随其后，显示出 Sonnet 系列作为“主力生产力工具”的极高采用率。

#### 2. MiniMax：流量之王，M2.5 霸榜
*   **数据异动**：`minimax/minimax-m2.5` 以 **1883.6B**（近 1.9 万亿）的 Token 消耗量位居全球第一，远超第二名 Gemini 3 Flash。
*   **分析**：这一惊人的流量数据表明 MiniMax 可能在特定应用场景（如角色扮演、长文本处理或大规模 B 端部署）取得了突破性进展，其模型调用量已达到现象级水平。

#### 3. Google：Gemini 3 Flash 策略奏效
*   **流量激增**：`google/gemini-3-flash-preview` 以 **1149.7B** 的消耗量排名第二。作为预览版模型，其流量甚至超过了正式版 Gemini 2.5，说明用户对“极速+低成本”的需求极其旺盛。
*   **跑分支撑**：Gemini 3 Flash 在 LMArena 中得分 **1473**，排名第七，证明其并未因速度而牺牲过多智能，属于“高性价比”首选模型。

#### 4. 中国厂商：DeepSeek 与 Kimi 稳健，智谱 GLM-5 进前十
*   **DeepSeek**：`deepseek-v3.2` 消耗量 **905.5B**，排名第三。DeepSeek 持续通过高性价比策略维持极高的市场份额。
*   **Moonshot (Kimi)**：`moonshotai/kimi-k2.5` 消耗量 **728.0B**，排名第五。Kimi 在长上下文场景的优势依然稳固。
*   **智谱 AI**：`z-ai/glm-5` 以 **674.4B** 的消耗量排名第八，且在 LMArena 跑分中达到 **1455** 分，排名第 16 位，是国产模型中跑分与流量双优的代表。

#### 5. OpenAI：GPT-5.2 面临压力
*   **跑分落后**：`gpt-5.2-chat-latest-20260210` 得分 **1481**，仅排名第六，已被 Claude Opus 4.6 和 Gemini 3.1 Pro 超越。
*   **流量隐忧**：在消耗量 Top 15 中，仅 `gpt-oss-120b`（疑似开源/轻量版）以 354.7B 排名第 14，主力模型 GPT-5.2 未出现在高消耗榜单中，可能意味着其市场份额正被 Claude 和 Gemini 侵蚀。
