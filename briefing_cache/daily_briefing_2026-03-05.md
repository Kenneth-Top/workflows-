# 【每日大模型行业情报简报】
**日期：2026年03月05日**

---

### 核心观点摘要
今日数据呈现出明显的**“头部洗牌”**与**“性价比博弈”**特征。
1.  **用量霸主易主**：MiniMax 的 `minimax-m2.5` 以 1.87 Trillion Tokens 的消耗量强势登顶，超越了 Google 和 DeepSeek，显示出极强的市场渗透力。
2.  **Claude 重夺技术王座**：Anthropic 的 `claude-opus-4.6` 以 1504 分重夺 LMArena 榜首，OpenAI 的 `gpt-5.2` 暂居第二梯队。
3.  **Google 闪存模型爆发**：`gemini-3-flash-preview` 用量激增至第 2 位，验证了市场对高性能低成本模型的巨大需求。

---

### 一、 核心厂商用量与格局分析

#### 1. MiniMax 异军突起，登顶消耗榜
*   **数据表现**：`minimax/minimax-m2.5` 近7日 Token 消耗量达到 **1870.07 B**，位居全榜第一。
*   **分析师点评**：这是非美系模型首次在单周消耗量上突破 1.8T 量级并登顶。考虑到 `minimax-m2.1` 同时也在榜单第 13 位，MiniMax 采用了“双模型”策略抢占市场。推测其近期可能进行了大规模 B 端部署落地或价格策略调整，成功承接了大量溢出需求。

#### 2. Google Gemini 3 系列全面铺开
*   **数据表现**：`gemini-3-flash-preview` 消耗量 **1232.56 B**（排名第2），远超旧版 `gemini-2.5-flash` 的 594.80 B。
*   **趋势研判**：Gemini 3 Flash 版本的消耗量几乎是上一代同期的两倍，表明 Google 在模型迭代速度和 API 推广上极其激进。结合其跑分（1473分），该模型已成为目前性价比最高的“主力生产模型”之一。

#### 3. 国产模型第一梯队稳固
*   **Moonshot (Kimi)**：`kimi-k2.5` 消耗量 **940.26 B**，稳居第 3 位。Kimi 在 C 端助手与 B 端 API 的双重驱动下，流量基本盘非常稳固。
*   **DeepSeek**：`deepseek-v3.2` 消耗量 **886.27 B**，排名第 4。尽管面临 MiniMax 的激烈竞争，DeepSeek 依然保持了极高的调用粘性，证明其技术口碑已转化为实际的生产流量。

#### 4. Anthropic 双雄并立，OpenAI 缺席前列
*   **Anthropic**：`claude-opus-4.6` (808.48 B) 与 `claude-sonnet-4.6` (778.62 B) 分别位列第 5、6 名。值得注意的是，旧版 `claude-sonnet-4.5` 流量已滑落至第 9 位，用户正快速向 4.6 版本迁移。
*   **OpenAI**：在消耗榜前 15 中，仅 `gpt-oss-120b` (412.90 B) 位列第 15 名。OpenAI 主力模型（如 GPT-5 系列）未出现在高频调用榜单前列，这可能意味着其流量更多集中在闭源生态或企业级私有部署中，未在公开 API 市场体现，或者其市场份额正受到开源/低成本模型的侵蚀。

### 二、 跑分与技术风向

根据 LMArena 最新排名（2026-03-05），**“Thinking”（思维链）模型已统治榜首**：

*   **Anthropic 领跑**：`claude-opus-4-6` 以 **1504 分**位居第一，其 Thinking 版本 `claude-opus-4-6-thinking` 紧随其后（1501分）。这标志着 Claude 在复杂推理任务上已超越 GPT 系列。
*   **Google 紧追不舍**：`gemini-3.1-pro-preview` 达到 **1500 分**，与榜首仅一步之遥。Google 的技术迭代速度令人咋舌，Gemini 3 系列已具备与 Claude Opus 正面对抗的实力。
*   **OpenAI 现状**：`gpt-5.2-chat-latest` 得分 **1480 分**，位列第 6。虽然仍属第一梯队，但相较于 Claude 和 Gemini 的最新模型，OpenAI 在公开跑分上的优势已不再明显。
*   **Grok 崛起**：xAI 的 `grok-4.20-beta1` 达到 1495 分，显示出马斯克旗下 AI 团队极强的工程能力。

### 三、 价格与成本分析

在 API 定价方面，**“极致压缩”**成为关键词：

*   **阿里通义**：`tongyi-deepresearch-30b-a3b` 的输入价格低至 **$0.089/M**，输出 **$0.449/M**。这是榜单中极具竞争力的低价，且拥有 71.8% 的缓存命中率，实际成本更低。阿里正在用价格战策略清洗中端模型市场。
*   **Amazon Nova**：`nova-lite-v1` 输入价格 **$0.059/M**，几乎击穿了行业底价。云厂商正在通过极低成本的“Lite”模型锁定长尾开发者。
*   **趋势总结**：头部厂商正在形成“双轨制”：旗舰模型（如 Claude Opus 4.6）维持高溢价与技术壁垒，而中轻量级模型（如 Gemini Flash, Tongyi, Nova）则通过近乎免费的价格抢占 Token 份额。

---

### 四、 多模态专区

虽然 LMArena 榜单主要展示文本得分，但结合定价数据与模型特性，我们发现以下多模态动向：

1.  **Vision 模型的平民化**：定价表中出现了 `allenai/molmo-2-8b`，输入价格仅 **$0.199/M**。Molmo 作为知名的多模态模型，其低价版本的推出意味着**视觉理解能力正在快速下沉**，不再是旗舰模型的专属。
2.  **Gemini 3 Flash 的多模态属性**：作为消耗榜第 2 名，`gemini-3-flash-preview` 原生支持多模态输入。其高达 1.2T 的消耗量暗示，**大量用户正在使用该模型进行图文混合处理**（如文档解析、截图理解），而非单纯的文本对话。
3.  **技术归因**：Claude Opus 4.6 与 Gemini 3.1 Pro 的高分跑分，很大程度上得益于其在多模态理解任务上的优化。随着多模态成为标配，纯文本模型的生存空间将被进一步压缩。

---

### 五、 风险提示与投资建议

*   **投资建议**：建议重点关注 **MiniMax** 与 **Anthropic** 的相关产业链机会。MiniMax 的流量爆发验证了其商业化落地能力；Anthropic 的技术领先地位稳固，且用户迁移意愿强（Sonnet 4.5 -> 4.6）。Google Gemini 系列的高性价比适合关注应用层降本逻辑。
*   **风险提示**：OpenAI 在公开 API 消耗榜上的“缺席”值得警惕。如果其流量未转移至私有部署，则可能面临市场份额被低价模型（DeepSeek, Gemini Flash）蚕食的风险。此外，行业价格战仍在持续，模型厂商的毛利率承压明显。
